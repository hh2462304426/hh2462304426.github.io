Tomcat 任意文件上传漏洞

漏洞编号

CVE-2017-12615

漏洞介绍

  • Tomcat运行在windows主机上，且启用了HTTP PUT请求方法，攻击者将有可能可通过精心构造的攻击请求向服务器上传包含的任意代码的JSP文件，之后，JSP文件中的代码被服务器执行。
  • 漏洞的产生是由于配置不当（非默认配置），将配置文件（conf/web.xml）中的readonly设置为了false，导致可以使用PUT方法上传任意文件。

影响版本

漏洞影响全部的 Tomcat 版本：Apache Tomcat 7.0.0 - 7.0.79 (windows环境)

漏洞复现

vulhub/tomcat/CVE-2017-12615

主页：http://your-ip:8080

抓包

冰蝎jsp木马，密码pass


使用冰蝎连接shell.jsp脚本

漏洞防护

  1. 配置readonly值为True或注释参数，禁止使用PUT方法并重启tomcat。
  2. 通过WAF阻止PUT和DELETE危险请求
  3. 静止用户上传JSP脚本

Apache Tomcat AJP文件包含漏洞

漏洞编号

CVE-2020-1938

漏洞介绍

  • Tomcat AJP（Apache JServ Protocol）：一种用于连接 Apache HTTP Server 和 Tomcat Web 容器的协议，AJP 协议主要用于将 Apache HTTP Server 作为反向代理服务器，将客户端的请求转发到运行 Tomcat 的后端服务器。
  • 攻击者通过 Tomcat AJP Connector 可以读取或包含 Tomcat 上所有 webapp 目录下的任意文件，例如可以读取 webapp 配置文件或源代码

影响版本

Tomcat 6.*
Tomcat 7.* < 7.0.100
Tomcat 8.* < 8.5.51
Tomcat 9.* < 9.0.31

漏洞复现

vulhub/tomcat/CVE-2020-1938

访问：http://your-ip:8080
poc下载（https://github.com/YDHCUI/CNVD-2020-10487-Tomcat-Ajp-lfi）
利用

Tomcat 弱口令+getshell

漏洞介绍

  • Tomcat服务自带一个管理界面，URL为：http://127.0.0.1:8080/manager/html
  • 在tomcat8默认安装下，tomcat没有任何用户，且该URL只允许本地的IP访问，那么这个漏洞就不存在了，但是如果业务有远程管理的需求，那么管理员就需要进行配置，如果管理员的用户密码配置不当，就可能被攻击者利用
  • 进入管理页面后，如果tomcat配置允许上传的权限，那么我们就可以上传war包，上传war包对于tomcat而言是一个模块，Tomcat 会将该war包解压，然后生成一个同名的目录，目录不是war压缩里的文件，这样，我们可以借助这个机制，上传jsp木马，连接成功后就可以获得shell权限了

漏洞复现

访问URL：http://127.0.0.1:8080/

使用：tomcat/tomcat

准备一个jsp木马，密码cmd

制作war包
